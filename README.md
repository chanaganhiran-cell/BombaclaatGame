💣Bombaclaat เป็นเกมแนว Turn-Based Tactical Board Game ที่ได้รับแรงบันดาลใจจากเกมอาร์เคดคลาสสิกอย่าง Bomberman แต่นำมาตีความใหม่ในรูปแบบการวางแผนสลับตาเดิน (Turn-based) 

ที่มาของชื่อเกม: "Bombaclaat" เป็นการเล่นคำกับสแลงจาเมกาที่มักใช้เป็นคำอุทานแสดงความตกใจสุดขีด ผสมผสานกับธีม "ระเบิด" (Bomb) ของเกม สื่อถึงความวายป่วงและการชิงไหวชิงพริบที่คุณต้องเจอ!

## 🎮 การควบคุม (Controls & Inputs)
เกมนี้ใช้การพิมพ์คำสั่งตัวอักษรแล้วกด `Enter` เพื่อทำแอ็กชันในเทิร์นของคุณ (ไม่จำกัดตัวพิมพ์เล็ก/ใหญ่):
* `W` : เดินขึ้น (Up)
* `S` : เดินลง (Down)
* `A` : เดินซ้าย (Left)
* `D` : เดินขวา (Right)
* `B` : วางระเบิด (Place Bomb)
* `O` : ข้ามเทิร์น / จบการกระทำในรอบนั้น (Skip/End Turn)
* `Q` : ออกจากเกม (Quit Game)

-------------------------------------------------------------------------------------------------------------------------------------------------

ในเกมนี้ ผู้เล่น 2-4 คนจะต้องใช้สมองในการบริหารแต้ม Action Point (AP) อ่านใจคู่ต่อสู้ และวางกับดักระเบิดเวลาบนตารางขนาด 9x9 เพื่อหาผู้เหลือรอดเพียงหนึ่งเดียว
🛠️ 1. การเตรียมกระดาน
    ขนาดกระดาน : ตารางขนาด 9x9 ช่อง
    สิ่งกีดขวาง : แบ่งออกเป็น 2 ชนิด
        - "#" Solid Wall : วางอยู่ที่ตำแหน่งแถวและคอลัมน์ที่เป็นเลขคู่สลับกัน เช่น (2,2), (2,4), (4,2) เป็นต้น กำแพงนี้ ไม่สามารถทำลายได้ และ เดินผ่านไม่ได้
        - "+" Breakable Wall : สุ่มวางในพื้นที่ว่างประมาณ 30-40% ของกระดาน สามารถทำลายได้ด้วยแรงระเบิด และมีโอกาสดรอปไอเท็ม
    จุดเริ่มต้น (Player Spawns): ผู้เล่นแต่ละคนจะเริ่มที่มุมสุดของแผนที่ทั้ง 4 มุม ได้แก่ (0,0),(0,8), (8,0) เเละ (8,8)

🎮 2. วิธีการเล่น การเล่นจะดำเนินไปเป็นรอบๆ โดยผู้เล่นแต่ละคนจะมี 2 แต้ม (Action Point - AP) ต่อรอบ สำหรับใช้เลือกทำแอ็กชันต่อไปนี้:
    🚶‍♂️ การเคลื่อนที่
        - ใช้ 1 AP ต่อการเดิน 1 ช่อง
        - เดินได้เฉพาะ 4 ทิศทาง: บน, ล่าง, ซ้าย, ขวา (เดินทแยงไม่ได้)
    💣 การวางระเบิด 
        - ไม่ใช้ AP (0 AP)
        - จำกัดการวางได้เพียง 1 ลูกต่อ 1 รอบ
        - ระเบิดจะถูกวางลงในช่องปัจจุบันที่ผู้เล่นยืนอยู่
    📝 หมายเหตุการจบเทิร์น
        - ผู้เล่นสามารถเลือก ข้ามเทิร์น หรือใช้ AP เพียง 1 แต้มแล้วจบเทิร์นได้ การจะเริ่มรอบถัดไป ผู้เล่นทุกคนจะต้องจบเทิร์นของตัวเองในรอบนั้นๆแล้ว

💥 3. กลไกการระเบิด (Explosion Mechanic)
    ⏱️ การนับถอยหลัง
         -ระเบิดจะมีอายุการใช้งาน 3 เทิร์น ตัวเลขจะลดลงหลังจากสิ้นสุดเทิร์นของผู้เล่นที่วางลูกนั้นๆ เมื่อเวลานับถอยหลังถึง 0 ระเบิดจะทำงานทันทีในช่วง ต้นเทิร์นถัดไป
    🔥 รัศมีการระเบิด:
        - ระเบิดออกเป็นรูปเครื่องหมายบวก (+) รัศมี 2 ช่องในทุกทิศทาง แรงระเบิดจะถูก หยุด โดย Solid Wall
        - แรงระเบิดจะ ทำลาย Breakable Wall ให้หายไปได้ (แรงระเบิดจะไม่ทะลุกำแพงอิฐไปโดนช่องหลังกำแพง)
    🔗 ปฏิกิริยาลูกโซ่ (Chain Reaction): 
        - หากแรงระเบิดไปโดนระเบิดอีกลูกที่วางอยู่บนกระดาน ระเบิดลูกนั้นจะทำงานทันที โดยไม่สนใจตัวเลขเวลานับถอยหลังที่เหลืออยู่

⚔️ 4. ความเสียหายและการโต้ตอบ (Damage & Interaction)
    กรณีระเบิดโดนผู้เล่น:
        - ถ้าผู้เล่นที่อยู่ในช่องรัศมีระเบิดแต่ไม่ใช่จุดศูนย์กลาง จะได้รับความเสียหาย HP -1
        - ศูนย์กลางระเบิด (Instant Kill): หากผู้เล่นอยู่บริเวณ "ใจกลาง" ของแรงระเบิดพอดี (ยืนทับช่องที่มีระเบิดในจังหวะที่ระเบิดทำงาน) จะถือว่า ตายและแพ้ในทันที
        - กรณีระเบิดโดนกำแพงอิฐ (Breakable Wall): กำแพงจะถูกทำลายหายไป และมีโอกาสดรอป ไอเท็มเสริม ให้ผู้เล่นเดินไปเก็บได้

🎁 5. ระบบไอเท็มเสริม
    เมื่อ Breakable Wall ถูกทำลาย จะมีโอกาสประมาณ 20-30% ที่จะเกิดไอเท็มปรากฏขึ้นบนช่องนั้น ผู้เล่นสามารถเก็บไอเท็มได้โดยการเดินไปทับช่องดังกล่าว
    +------------------+----------------------------------------------------------------------------------------+------------+
    | ไอเท็ม            | ความสามารถ                                                                             | ซ้อนทับกันได้ |
    +------------------+----------------------------------------------------------------------------------------+------------+
    | 🧨 Fire Up (F)  | ขยายรัศมีการระเบิดของระเบิดลูกถัดๆ ไปที่ผู้เล่นวาง +1 ช่อง (เช่น จากรัศมี 2 เพิ่มเป็น 3, 4, 5...)          |   ✅ ได้   |
    | 👟 Speed Up (S) | เพิ่มแต้มการกระทำ (Action Point) +1 AP ถาวร ในทุกๆ รอบ                                      |   ✅ ได้   |
    | 💣 Bomb Up (B)  | อนุญาตให้ผู้เล่นมีระเบิดที่กำลังทำงานอยู่บนกระดานได้สูงสุด เพิ่มขึ้น 1 ลูก (จากเดิม 1 ลูก เป็น 2 ลูกพร้อมกัน)   |   ✅ ได้   |
    +----------------+------------------------------------------------------------------------------------------+------------+

🏆 6. การตัดสินแพ้ชนะ (Win/Loss Conditions)
    👑 ผู้ชนะ (Winner): คือผู้เล่นที่เหลือรอดชีวิตเป็นคนสุดท้าย บนกระดาน
    💀 ผู้แพ้ (Loser): คือผู้เล่นที่ถูกแรงระเบิดจน HP หมด หรือเดินเข้าไปในรัศมีระเบิดที่กำลังทำงานจนตาย
    🤝 ผลเสมอ (Draw): จะเกิดขึ้นก็ต่อเมื่อ ระเบิดลูกเดียว หรือโดนระเบิดในเทิร์นเดียวกัน ทำลายผู้เล่น 2 คนสุดท้ายที่เหลืออยู่พร้อมกันในเทิร์นนั้น

💻 1. แพลตฟอร์มที่รันเกม (Target Platform)
    เกมนี้ถูกออกแบบมาให้รันบน Console / Terminal (Command Line Interface) ครับ โดยสามารถรันข้ามแพลตฟอร์ม (Cross-Platform) ได้หลักๆ 2 ระบบปฏิบัติการ:
    - Windows: ผ่านโปรแกรมอย่าง Command Prompt, PowerShell หรือ Terminal ใน VS Code
    - Unix-like (Linux / macOS): ผ่านโปรแกรม Terminal ต่างๆ (อ้างอิงจากการใช้คำสั่ง system("clear") ในโค้ด #else)
    เนื่องจากเกมใช้คำสั่ง system() ในการล้างหน้าจอ และใช้ cin ในการรับค่า มันจึงเป็นเกมประเภท Turn-based Text Game ที่ต้องพิมพ์ตัวอักษรแล้วกด Enter เพื่อให้เกมทำงานไปทีละสเต็ปครับ
🛠️ 2. เครื่องมือและไลบรารีที่ใช้ (Tech Stack & Libraries)
    ภาษาโปรแกรมหลัก (Core Language):
    - C++ โดยใช้แนวคิดการเขียนโปรแกรมเชิงวัตถุ (Object-Oriented Programming - OOP) มี Class, Inheritance, Polymorphism ผ่าน virtual function
    - ไลบรารีมาตรฐานที่ใช้งาน (C++ Standard Libraries):
    - เกมนี้ไม่ได้ใช้ Game Engine หรือไลบรารีภายนอกเลย แต่พึ่งพาไลบรารีมาตรฐานของ C++ ล้วนๆ ดังนี้:
    <iostream> หน้าที่: จัดการการนำเข้าและส่งออกข้อมูลพื้นฐาน
        ตัวอย่างการใช้ในเกม: ใช้ cout สำหรับวาดตาราง/ข้อความโต้ตอบ และ cin สำหรับรับคำสั่งควบคุมตัวละครจากผู้เล่น (W, A, S, D, B)
    <vector> หน้าที่: โครงสร้างข้อมูลแบบ Dynamic Array (อาเรย์ที่ยืดหดขนาดได้)
        ตัวอย่างการใช้ในเกม: ใช้เก็บและจัดการลิสต์ของผู้เล่น (vector<Player*> players), ระเบิดที่วางอยู่บนกระดาน (vector<Bomb*> activeBombs), และไอเท็มที่ดรอป (vector<Item*> items)
    <string> หน้าที่: จัดการข้อมูลประเภทข้อความ
        ตัวอย่างการใช้ในเกม: ใช้เก็บชื่อของผู้เล่น (เช่น "Player 1", "Player 2")
    <cstdlib> หน้าที่: ไลบรารีอรรถประโยชน์ทั่วไป (General purpose utilities)
        ตัวอย่างการใช้ในเกม: ใช้คำสั่ง rand() สำหรับสุ่มตัวเลข (เช่น สุ่มเกิดกำแพงอิฐ, สุ่มดรอปไอเท็ม) และฟังก์ชัน system() สำหรับล้างหน้าจอ
    <ctime> หน้าที่: จัดการข้อมูลเกี่ยวกับเวลา
        ตัวอย่างการใช้ในเกม: ใช้คู่กับ <cstdlib> โดยเรียก time(0) มาเป็น Seed ให้กับฟังก์ชัน srand() เพื่อให้การสุ่มตัวเลขในเกมไม่ซ้ำกันในแต่ละครั้งที่เปิดเกมใหม่
    <iomanip> หน้าที่: จัดรูปแบบการแสดงผล (I/O Manipulation)
        ตัวอย่างการใช้ในเกม: ใช้คำสั่ง left และ setw(10) ในฟังก์ชัน display() เพื่อจัดให้ข้อความ Status ของผู้เล่น (เช่น ชื่อและ HP) เรียงตรงกันเป็นคอลัมน์สวยงามเมื่อแสดงผล

FUNCTIONAL REQUIREMENTS
 - ระบบต้องสร้างกระดานตารางขนาด 9x9 ช่องเมื่อเริ่มเกมใหม่
 - ระบบต้องวางผู้เล่น 2 ถึง 4 คน ในตำแหน่ง (0,0), (0,8), (8,0) และ (8,8) ตามลำดับ
 - ระบบต้องวาง Solid Wall (#) แบบตายตัวในตำแหน่งแถวและคอลัมน์ที่เป็นเลขคู่
 - ระบบต้องสุ่มวาง Breakable Wall (+) ในพื้นที่ว่างของกระดานในอัตรา 30-40%
 - ระบบต้องมอบ Action Point (AP) จำนวน 2 แต้มให้ผู้เล่นแต่ละคนเมื่อเริ่มต้นเทิร์นของตนเอง
 - ระบบต้องตัด 1 AP เมื่อผู้เล่นสั่งเดิน 1 ช่อง (รองรับเฉพาะทิศทาง บน, ล่าง, ซ้าย, ขวา)
 - ระบบต้องบล็อกการเดิน หากช่องเป้าหมายคือ Solid Wall, Breakable Wall หรืออยู่นอกขอบกระดาน
 - ระบบต้องอนุญาตให้ผู้เล่นวางระเบิด ณ ตำแหน่งปัจจุบัน 1 ลูกต่อเทิร์น โดยไม่หัก AP
 - ระบบต้องมีฟังก์ชันให้ผู้เล่นเลือกจบเทิร์น (Skip) ได้ทันทีแม้ AP จะยังเหลืออยู่
 - ระบบต้องเปลี่ยนรอบการเล่น (Round) ก็ต่อเมื่อผู้เล่นทุกคนในรอบนั้นจบเทิร์นแล้ว
 - ระบบต้องนับถอยหลังอายุของระเบิด 3 เทิร์น และสั่งให้ระเบิดทำงานในต้นเทิร์นถัดไปเมื่อเวลาถึง 0
 - ระบบต้องสร้างรัศมีความเสียหายเป็นรูปเครื่องหมายบวก (+) ระยะ 2 ช่อง
 - ระบบต้องหยุดการขยายรัศมีระเบิดทันทีหากรัศมีชนเข้ากับ Solid Wall
 - ระบบต้องทำลาย Breakable Wall ที่โดนรัศมีระเบิดให้หายไป และไม่ทะลุไปโดนช่องด้านหลังกำแพง
 - ระบบต้องสั่งให้ระเบิดลูกอื่นทำงานทันที (Chain Reaction) หากโดนรัศมีระเบิด โดยไม่สนเวลาที่เหลือ
 - ระบบต้องลด HP ของผู้เล่น 1 แต้ม หากผู้เล่นยืนอยู่ในรัศมีระเบิด (ที่ไม่ใช่จุดศูนย์กลาง)
 - ระบบต้องปรับ HP ผู้เล่นเป็น 0 (ตายทันที) หากผู้เล่นยืนอยู่บนจุดศูนย์กลางของระเบิดที่กำลังทำงาน
 - ระบบต้องสุ่มเกิดไอเท็มบนช่องกระดานด้วยโอกาส 20-30% ทันทีที่ Breakable Wall ถูกทำลาย
 - ระบบต้องลบไอเท็มออกจากกระดาน และมอบบัฟให้ผู้เล่นทันทีเมื่อผู้เล่นเดินทับช่องไอเท็มนั้น
 - ระบบต้องเพิ่มรัศมีระเบิด +1 ช่อง ต่อไอเท็ม Fire Up (F) ที่ผู้เล่นเก็บได้ (สามารถซ้อนทับได้)
 - ระบบต้องเพิ่ม AP สูงสุด +1 แต้ม ต่อไอเท็ม Speed Up (S) ที่ผู้เล่นเก็บได้ (สามารถซ้อนทับได้)
 - ระบบต้องเพิ่มขีดจำกัดการวางระเบิด +1 ลูก ต่อไอเท็ม Bomb Up (B) ที่ผู้เล่นเก็บได้ (สามารถซ้อนทับได้)
 - ระบบต้องแสดงผลประกาศแพ้-ชนะ-เสมอ ตามเงื่อนไขการเหลือรอดบนกระดานอย่างถูกต้อง
NON-FUNCTIONAL REQUIREMENTS
 - ระบบต้องแสดงผลสัญลักษณ์บนกระดาน (เช่น #, +, F, S, B) ให้มีความแตกต่างกันอย่างชัดเจน เพื่อให้ผู้เล่นอ่านสถานการณ์ได้โดยไม่สับสน
 - ระบบต้องแสดงสถานะปัจจุบันของผู้เล่น (เช่น HP ที่เหลือ, AP ที่เหลือในเทิร์น, จำนวนระเบิดที่วางได้) ให้ผู้เล่นทราบอย่างชัดเจนในทุกๆ การกระทำ
 - ระบบต้องสามารถรับมือกับการกดคำสั่งที่ผิดพลาด (Error Handling) ของผู้เล่นได้ เช่น พยายามเดินทะลุกำแพง โดยไม่ทำให้เกมล่มหรือเกิดบั๊ก
 - การคำนวณปฏิกิริยาลูกโซ่ (Chain Reaction) ของระเบิดหลายลูกพร้อมกันบนกระดาน ต้องประมวลผลให้เสร็จสิ้นอย่างรวดเร็ว โดยไม่เกิดอาการหน่วงหรือแลค(Lag)
 - กลไกของเกมต้องถูกออกแบบมาให้รองรับการสลับเทิร์นได้อย่างไหลลื่น ไม่ว่าจะมีผู้เล่น 2, 3 หรือ 4 คนก็ตาม
 - อัลกอริทึมการสุ่ม (RNG) สำหรับการวาง Breakable Wall และการสุ่มดรอปไอเท็ม ต้องมีการกระจายตัวแบบสุ่มอย่างแท้จริง เพื่อไม่ให้เกิดความได้เปรียบเสียเปรียบ
